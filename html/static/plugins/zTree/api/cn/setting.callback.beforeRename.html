Function(treeId, treeNode, newName, isCancel)setting.callback.beforeRename

Overview[ Dependency jquery.ztree.exedit Extension js ]

For captureNodesEdit NameEnd（Input Lost focus 或 Press Enter 键）After，UpdateNode NameData之前的EventsRevert function，并且Based onBack值Sure是否AllowChange名称的Operation

Nodes进入Edit NameStatus后，按 ESC 键可以放弃当前Modify，Restore原名称，CancelEdit NameStatus

从 v3.5.13 Start，Cancel编辑Status也会Trigger此回调，Based on isCancel ParametersDecision

Default value：null

Function Parameters说明

treeIdString

Correspond zTree 的 treeId，便于User操控

treeNodeJSON

将要Change名称的Nodes JSON DataObject

newNameString

Modify后的新名称

isCancelBoolean

是否CancelOperation (v3.5.13+)

isCancel = true OrganisationCancel编辑Operation（Press ESC 或 Use cancelEditName Methodology）

isCancel = false OrganisationConfirmModifyOperation

Back值Boolean

Back值是 true / false

IfBack false，zTree 将Hold名称编辑Status，无法Trigger onRename EventsRevert function，并且会导致屏蔽其它Events，直到Modify名称使得 beforeRename Back true

IfBack false，I won\'t input 输入框Access焦点，避免Because警告Information而导致反复Trigger beforeRename。 请在CloseHint警告Information后，Utilization editName Methodology让 input 重新Access焦点。

}
