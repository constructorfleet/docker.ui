Function(event, treeId, treeNode, isCancel)setting.callback.onRename

Overview[ Dependency jquery.ztree.exedit Extension js ]

For captureNodesEdit NameEndAfter的EventsRevert function。

1、NodesEnterEdit NameStatus，并且ModifyNode Name后Trigger此Revert function。IfUserSettings了 beforeRename Revert function，并Back false，将无法Trigger onRename EventsRevert function。

2、IfPass直接Modify treeNode 的Data，并且Utilization updateNode MethodologyUpdate，是不会Trigger此Revert function的。

3、从 v3.5.13 Start，Cancel编辑Status也会Trigger此回调，Based on isCancel ParametersDecision

Default value：null

Function Parameters说明

eventjs event Object

Standard的 js event Object

treeIdString

Correspond zTree 的 treeId，便于User操控

treeNodeJSON

被Modify名称的Nodes JSON DataObject

isCancelBoolean

是否CancelOperation (v3.5.13+)

isCancel = true OrganisationCancel编辑Operation（Press ESC 或 Use cancelEditName Methodology）

isCancel = false OrganisationConfirmModifyOperation

setting & function Example

1. Modify名称后，弹出被Modify名称的Nodes的 tId and name Information

function zTreeOnRename(event, treeId, treeNode, isCancel) {
	alert(treeNode.tId + ", " + treeNode.name);
}
var setting = {
	callback: {
		onRename: zTreeOnRename
	}
};
