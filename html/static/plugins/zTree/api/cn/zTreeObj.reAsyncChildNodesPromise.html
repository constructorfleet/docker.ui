Function(parentNode, reloadType, isSilent)zTreeObj.reAsyncChildNodesPromise

Overview[ Dependency jquery.ztree.core Core js ]

强行Step Load父Nodes的子Nodes（ES6 Promise 版）。[setting.async.enable = true It works]

已经加载过的父Nodes可反复Use此MethodologyReload。

请Pass zTree Object执行此Methodology。

v3.5.29+

Function ParametersAnnotations

parentNodeJSON

指定YesStep Load的父Nodes JSON Data

1、parentNode = null 时，equivalent to从根Nodes Root ConductStep Load

2、parentNode.isParent = false 时，不ConductStep Load

3、请务必Promise此NodesDataObject 是 zTree 内部的DataObject

reloadTypeString

reloadType = "refresh" OrganisationClear后Reload。

reloadType != "refresh" 时，OrganisationAppend子Nodes处理。

isSilentBoolean

设定Step Load后是否AutoExpand父Nodes。

isSilent = true 时，不Expand父Nodes，Other值或缺省Status都AutoExpand。

Back值ES6 Promise

ES6 的 Promise Object，便于异步Control，Use时请AttentionBrowser兼容问题

function Example

1. 重新Step Load zTree 

var treeObj = $.fn.zTree.getZTreeObj("tree");
treeObj.reAsyncChildNodes(null, "refresh");

2. 重新Step Load当前选Medium第一个Nodes

var treeObj = $.fn.zTree.getZTreeObj("tree");
var nodes = treeObj.getSelectedNodes();
if (nodes.length>0) {
	treeObj.reAsyncChildNodes(nodes[0], "refresh");
}
